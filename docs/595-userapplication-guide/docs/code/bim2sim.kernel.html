Combine IfcOpenShell file instance, finder and units together.

Especially if we handle multiple IFC files this dataclass helps us to keep track which finder and units belong to which ifc file.

Parameters:

  • ifc_path (Path) – Pathlib object that points to ifc file

  • reset_guids (bool) – Boolean that determine if GUIDs should be reset

  • ifc_domain (Optional[IFCDomain]) – Domain of the given ifc file if this is known

get_ifc_units()¶

Returns dict to translate IFC units to pint units

To use units from IFC we get all unit definitions from the ifc and their corresponding measurement instances and map them to pint units.

Return type:

dict where key is the IfcMeasurement and value the pint unit definition. e.g. ‘IfcLengthMeasure’: meter

format(record)¶

Format the specified record as text.

The record’s attribute dictionary is used as the operand to a string formatting operation which yields the returned string. Before formatting the dictionary, a couple of preparatory steps are carried out. The message attribute of the record is computed using LogRecord.getMessage(). If the formatting string uses the time (as determined by a call to usesTime(), formatTime() is called to format the event time. If there is exception information, it is formatted using formatException() and appended to the message.
